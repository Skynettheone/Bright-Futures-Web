Home Page

Technical Description

The Home Page is structured with semantic HTML5 tags to create a logical and organized document. The primary sections include a <header> for navigation, several <section> elements for distinct content areas like the hero banner and content cards, and a <footer> for concluding information.

The styling is handled by an external stylesheet (home.css), which uses modern CSS techniques for layout and aesthetics:

  • Layout and Positioning: The hero section (.hero) utilizes display: flex to center content vertically. It employs position: absolute for the background video (.hero-video) to layer it behind the text content, which has a higher z-index. The "Clean Energy Solutions" area uses display: grid on the .card-container to create a responsive four-column layout.
  • Styling and Visual Effects: The background video has a filter: brightness(0.7) applied to darken it, ensuring the overlay text is legible. linear-gradient is used for smooth background transitions. Interactive elements feature hover effects using the transform: translateY() and box-shadow properties to provide visual feedback.

Accessibility

  • Semantic HTML: The page structure is built with semantic tags like <header>, <nav>, <section>, and <footer>. This provides an immediate, clear structure for screen readers and other assistive technologies.
  • Alternative Text: All <img> tags include descriptive alt attributes, such as "Logo" and "SDG Goal 7 Logo," to convey their meaning to users who cannot see them.
  • ARIA Labels: To enhance clarity, aria-label attributes are used where context is needed. The background video has an aria-label="Background video showing renewable energy sources", and social media icons have labels like aria-label="Facebook".
  • Logical Heading Structure: The page follows a logical heading hierarchy, starting with an <h1> and using <h2> for subsequent major sections. This allows users to easily navigate the document's outline.
  • Keyboard Navigability: All interactive elements, primarily anchor tags (<a>), are natively focusable, ensuring that users can navigate the entire page using only a keyboard.

Data Page

Technical Description

The Data Page is built around <table> element, which is the appropriate choice for displaying structured, tabular data. The table is organized with a <thead> for column headers and a <tbody> for the country data. The colspan attribute is used in the header rows to merge cells, creating a clear categorical structure for the data presented.

The styling from data.css enhances the table's readability and visual appeal:

  • Layout and Styling: The table headers (.table-header) and the page title (.article-header) use a linear-gradient for a strong visual identity. A subtle box-shadow on the table (.energy-data-table) lifts it from the page. Status indicators are styled with distinct background colors (e.g., .status-red) to provide a quick visual reference.
  • Interactive Tooltips: A key feature is the custom CSS-only tooltip on the table headers. This is implemented using a [data-tooltip] attribute selector and a ::before pseudo-element. The pseudo-element is absolutely positioned relative to the header cell to display the tooltip text on hover, providing users with extra context for each column.

Accessibility

  • Semantic Table Structure: Using the <table> element with <thead>, <tbody>, <th>, and <td> tags is the most critical accessibility feature here. This structure allows screen readers to correctly interpret the relationships between headers and data cells, so a user can navigate the data logically.
  • Header Tooltips and Link Titles: The data-tooltip attribute on each header cell provides descriptive text that explains the column's purpose. Similarly, the <a> tags for external policies include a title attribute, offering more context about the link's destination.
  • Avoiding Color-Only Information: While status indicators use color (green, yellow, red) for quick reference, they are not a substitute for text. Crucially, each indicator also contains descriptive text (e.g., "Stagnant progress on energy intensity..."). This ensures that users who are color-blind or cannot perceive color can still understand the information.
  • Alternative Text: The SDG logo image within the table header has a descriptive alt="SDG 7 Logo" attribute, ensuring its purpose is conveyed to non-visual users.

Content Page

Technical Description

This page is structured using several <article> tags to organize the content into logical sections. The layout for each section is controlled with CSS Flexbox.

  • Flexbox Layout: A key class, .article-layout, uses display: flex to place text and images side-by-side. An additional .reverse class is used to alternate the order, creating an engaging, varied layout down the page.
  • Sticky Navigation: The page features a navigation bar (.content-nav) that uses position: sticky. This keeps the menu fixed to the top of the screen as the user scrolls, allowing for easy navigation between article sections.

Accessibility

  • In-Page Navigation: The sticky navigation bar contains links like <a href="#intro"> which correspond to elements with an id="intro". This creates a clear document outline and allows users, especially those with screen readers, to easily jump between sections.
  • Alternative Text: Every image on the page includes a descriptive alt attribute (e.g., alt="Improved cookstove in rural household"). This provides essential context for users who cannot see the images.
  • Logical Headings: The page uses a logical heading structure (<h1> for the main title and <h2> for section titles), which helps create a clear and navigable structure for all users.

Challenges and Lessons Learned

This project was a great learning experience that pushed me to solve several common web development problems.

1. Implementing the Sitewide Tagline Banner

Challenge: I wanted to add the "17 Goals · One Planet · Our Future" banner above the main navigation on every single page. The challenge was making it a consistent, separate element that didn't interfere with the primary navigation bar.

Solution: I structured the HTML by placing the banner's `div` within the main <header> tag but just before the <nav> element. This allowed me to style it as its own distinct bar while keeping it part of the overall header. To ensure consistency, I reused this exact block of HTML across all pages.

Lesson Learned: This taught me how important a good HTML structure is. By planning the header layout properly, I could easily add new elements later without having to redo the CSS.

2. Sticky Navigation and Page Anchors

Challenge: On the content page, whenever I clicked a link to jump to a section, the section's title would get hidden behind the sticky navigation bar.

Solution: I solved this with JavaScript. I wrote a simple script with help of AI that intercepts the click, finds the position of the target section, and then scrolls the page to a point slightly above it, leaving enough space so the title is perfectly visible below the sticky nav.

Lesson Learned: This was a practical lesson in how CSS features and JavaScript interactivity are connected. Sometimes a problem created by a CSS property needs a JavaScript solution to ensure a good user experience.

3. Time Management and Staying Focused

Challenge: Juggling three unique pages and overall page consistency, each with its own content and layout requirements, was more time-consuming than I first expected.

Solution:I prioritized building the core structure and functionality first before getting lost in minor design tweaks. It helped me allocate my time more evenly across the project.

Lesson Learned: Project and time management are skills just as important as coding.
